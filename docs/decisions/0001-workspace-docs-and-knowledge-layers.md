# 0001. Workspace docs and knowledge layers

- Status: Accepted
- Date: 2026-07-08

## Context

Knowledge about this workspace is scattered across several places: per-project
`README.md` / `docs/`, ephemeral notes in `_docs/` (worklogs, handoffs, session
registry), skill-generated artifacts in `docs/superpowers/`, Claude Code
auto-memory, and — as of 2026-07-08 — a personal cross-project wiki at `~/wiki`
(a separate private repo). Without an explicit rule it is unclear where a given
piece of knowledge belongs, so notes get duplicated or lost, and `_docs/` tends
to grow into a dumping ground.

Separately, git log already records *what* changed and *when*. What it cannot
recover is the *why* behind a decision — rejected alternatives and decisions
that produce no diff.

## Decision

Define where each kind of knowledge lives:

- **Per-project `docs/` and `README.md`** — specs, design, and product docs
  specific to that one project. The source of truth for that project.
- **`docs/decisions/` (this directory)** — ADRs for *workspace-level*
  load-bearing decisions whose rationale git log can't recover. Thin by design.
- **`_docs/`** — ephemeral flow docs (worklogs, handoffs, session registry).
  Not curated; may be pruned. Listed under "Do Not Inspect By Default".
- **`docs/superpowers/`** — plans/specs generated by skills. Generated
  artifacts, not hand-authored docs.
- **`~/wiki`** — personal, cross-project *durable* knowledge (reusable lessons,
  environment notes). Separate private repo; see its `AGENTS.md`.
- **git log** — the "what/when" history. ADRs only fill what the log can't.

Write an ADR only when a future reader can't reconstruct the "why" from the
diff/log. Otherwise leave it to git log.

## Consequences

- Each kind of knowledge has a clear home; less duplication and drift.
- ADRs stay load-bearing-only and therefore few and readable.
- No CHANGELOG / releases / runbooks / glossary ceremony until there is an
  actual reader that needs it.
- Cross-project reusable lessons are distilled to `~/wiki`, not duplicated here.
