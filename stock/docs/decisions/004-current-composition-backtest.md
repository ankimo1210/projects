# ADR 004: バスケットは現在構成銘柄を過去全期間に適用する

**Status**: Accepted
**Date**: 2026-05-19

## Context

インデックスバスケット計算では、過去のリターンを再現する際に「どの時点の構成銘柄を使うか」を決める必要がある。

選択肢：
1. **現在の構成銘柄を全期間に適用** — 簡単、ただし survivorship bias
2. **時点別の構成銘柄を使用** — 正確だが履歴データ取得が必要

## Decision

**(1) 現在の構成銘柄を過去全期間に適用** する（暫定）。バスケットの解釈を「現在の構成銘柄を◯年前から保有していたら」というシミュレーションと位置付け、ドキュメントに明示する。

## Rationale

### (2) を見送った理由

- **歴史的構成銘柄リストの入手コスト** — 公式 API は有料、Wikipedia は変更履歴があるがパース複雑
- **段階的に改善可能** — まず基本機能を出して、後から精度を改善できる

### (1) でも価値がある理由

- **現在のセクター/銘柄寄与度分析** には十分
- **個別銘柄のウェイト推移** が見られる（公式指数では見られない）
- **短期 (3ヶ月以内) の指数追随** は誤差が小さい

## Consequences

- **バスケット ≠ 公式指数リターン** — 系統的な乖離（DJIA +7%, NDX +14% 等）
- **ETF を「正確なベンチマーク」として併記** — UI上で常に表示
- **解釈の明示** — METHODOLOGY.md で「これは何のシミュレーションか」を文書化

## Future Improvements

| 改善案 | 効果 | 実装難度 |
|---|---|---|
| Wikipedia の入替履歴をパース | 大幅改善 | 中 |
| 公式 methodology PDF を読み込み | 完全再現 | 高（OCR必要）|
| 構成銘柄スナップショットの定期保存 | 将来分の改善 | 低（cronで取得）|
