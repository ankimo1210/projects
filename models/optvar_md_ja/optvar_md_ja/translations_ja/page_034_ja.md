3.6      数値実装

このセクションでは、Opt-Varモデルの生産環境での実装と使用されたプラットフォームおよびパッケージについて説明します。
3.6.1     Java でのモデル実装

Opt-Varモデルは二次最適化問題として表現され、Javaで生産環境に実装されています。具体的にはNAG最適化ライブラリを使用し、特にE04NQルーチンを用いています。このルーチンはFortranのE04NQFルーチンをラップしており、二次最適化問題を解くのに適しています。

このルーチンは以下のパラメータを受け取り、これらは生産環境での実装で明示的に設定されています：
  + 最適性許容度：最適性許容度はデフォルト値10^-1に設定されています。許容度を小さくすると、反復回数が増加し、より-optimalな結果を得ることができます。

実装では、初期可行点も提供しています。最終的な上限と下限取引制限の中央点を選択して初期点として使用し、開始可能な初期点を確保します。最終的な上限と下限取引制限は、ポジション増加が許可されている場合 Eq. (7) から得られ、ポジション増加が許可されていない場合 Eq. (8a) および Eq. (8b) から得られます。

3.6.2     モデルの再構成

NAG E04NQ ルーチンに合わせて Java で Opt-Varモデルを再構成する必要があります。二次最適化問題を実装するために、以下の制約条件を考慮します：
  - 最小化 \((q + u)'(AD + b_0P_a)(q^T + u) + C_u' + u'Mu - a_l(q + u)\)
  - 制約：\(-H_y < S_0(q_i + u_i) < H_u, i = 1\)

ここで、\(S_y\) と \(S_u\) はそれぞれ取引量 \(u\) の最終的な下限と上限制約を表します；ポジション増加が許可されている場合 Eq. (7) から読み取り、ポジション増加が許可されていない場合符号によって異なる値になります。

問題を具体的に二次計画問題として再構成し、線形制約を持つようにすることで数値実装を行います。特に、オブジェクト関数 F の中にある定数は解に影響を与えませんので計算から省略できます。また、絶対値 |u| を追加の状態変数 z による線形項に置き換えることで二次計画問題を得ます。問題は以下のようになります：
  - 最小化 \(u^T (AD + b_0P_a) u + \frac{1}{2} u^T M u - a_l u\)
  - 制約：\(z_i > u_i, i \in [d]\)
