## 1. コンセプト（確定）

暗い宇宙空間を舞台に、発光する一本のライン（自機／トレイルを引くコメット）を操作し、
次々と迫ってくる障害物の**隙間をくぐり抜けて**どこまでスコアを伸ばせるかを競う、片手で遊べる無限ランゲーム。

- ジャンル: 隙間くぐり系ライン操作 / 無限ラン（エンドレス）
- 対象: カジュアル、スキマ時間、片手プレイ
- 中毒性の源泉: シンプル操作 + じわじわ上がる難易度 + ハイスコア更新欲

---

## 2. コアゲームプレイ（確定）

### 操作
- **タップ中だけ上昇。指を離すと重力で降下**（Flappy系の連続版）。
- 片手・縦持ちで完結。タップの押し込み時間で高さを微調整する感覚。

### 画面と進行
- **縦持ち（Portrait）固定**。
- 自機は画面左寄りに固定、背景と障害物が右→左へ流れる**横スクロール**。
- **無限ラン**: 自動で進み続け、進行距離ベースでスコアが伸びる。

### 失敗ルール（ライフ制）
- **ライフ3**。障害物に接触すると1ライフ消費。
- 接触直後は短い**無敵時間（例: 1.0〜1.5秒・点滅表示）**を与え、連続ミスを防ぐ。
- ライフ0でゲームオーバー → スコア／ハイスコア／リトライ表示。

### スコア
- 基本: 進行距離（時間）に応じて加算。
- ボーナス: 道中の**コイン**取得で加算。
- ハイスコアはローカル保存（後述）。

### 難易度ランプ（時間／距離で上昇）
- スクロール速度が徐々に上がる。
- 隙間が徐々に狭くなる。
- 障害物の出現密度が上がる。
- 上限（キャップ）を設け、理不尽にならない範囲で頭打ちにする。

## 3. 障害物設計（混合・段階的に多彩化）

進むほど新種が混ざる。数値は実装時の初期値・要チューニング。

| フェーズ | 目安 | 障害物 |
|---|---|---|
| Phase 1 | 序盤 | 上下から伸びる**静的な壁の隙間**（Flappy定番） |
| Phase 2 | 中盤 | **上下に動く壁**／隙間位置がランダム |
| Phase 3 | 後半 | **回転バー・回転ゲート**（隙間が回る） |
| Phase 4 | 終盤 | **複合**（動く＋回転、複数同時） |

- コインは隙間の近くやリスクのある位置に置き、「安全に抜ける vs コインを狙う」の判断を作る。
- 障害物とコインは画面右外で生成し、左外に出たら破棄（オブジェクトプール推奨）。

## 4. テーマ／ビジュアル（確定: ネオン／宇宙風）

- 暗い（ほぼ黒／濃紺）背景 + 発光するネオンのライン・図形。
- 自機はトレイル（残像）を引くコメット風。`SKEmitterNode` でパーティクル。
- 障害物はネオンの輪郭（グロー）で描画。最初は図形ベース（`SKShapeNode`）でOK、画像アセットは後回し可。
- 配色は少数のネオン色（シアン／マゼンタ／ライム等）を背景の暗色に映えさせる。
- コイン取得・ミス時にグロー明滅やパーティクルで手ごたえを出す。

## 6. 技術方針（確定）

- **SwiftUI アプリシェル + SpriteKit（ゲーム本体）**。`SpriteView` で `SKScene` を埋め込む。
- ゲームループは `SKScene.update(_:)`。物理・衝突は SpriteKit の `SKPhysicsBody` を使用。
- **衝突判定はカテゴリビットマスク**で管理（player / obstacle / coin / worldBounds）。
  - 接触検知は `SKPhysicsContactDelegate`。障害物ヒットはライフ減、コインは取得処理。
- **永続化は UserDefaults**（ハイスコア、累計コイン程度）。
  - ※前回（えいたんクエスト）はSwiftDataだったが、本作は保存対象が少数のためUserDefaultsで十分。SwiftDataはオーバースペックなので使わない。
- **サウンド**: 短い効果音は `SKAction.playSoundFileNamed`、BGMを入れる場合は `AVAudioPlayer`。
- **ハプティクス**: `UIImpactFeedbackGenerator`（軽量）。凝るなら `CoreHaptics`。
- **Deployment Target: iOS 17.0**（前回と揃える）。
- 端末向き: **Portrait固定**。
- 依存: 原則Apple標準フレームワークのみ（外部ライブラリなし）。

## 7. 画面／シーン構成

```
NeonThreadApp (SwiftUI @main)
└─ ContentView (SwiftUI)
   ├─ TitleView            … タイトル / ハイスコア表示 / Start
   ├─ GameView             … SpriteView(scene: GameScene) をホスト
   │                          HUD（スコア・ライフ・コイン）はSwiftUIオーバーレイ or SKNode
   └─ GameOverView         … 今回スコア / ハイスコア / リトライ / タイトルへ

SpriteKit
└─ GameScene : SKScene
   ├─ Player (自機ノード + トレイル emitter, physicsBody)
   ├─ ObstacleSpawner (フェーズに応じて生成、プール管理)
   ├─ CoinSpawner
   ├─ ScrollController (速度・難易度ランプ)
   └─ PhysicsContactDelegate (衝突→ライフ/コイン処理)
```

画面遷移はシンプルな状態（enum: title / playing / gameOver）で管理。

## 8. データモデル / 状態

### 保存（UserDefaults）
| キー | 型 | 内容 |
|---|---|---|
| `highScore` | Int | ベストスコア |
| `totalCoins` | Int | 累計コイン（任意） |
| `isSoundEnabled` | Bool | サウンドON/OFF（任意） |
| `isHapticsEnabled` | Bool | 振動ON/OFF（任意） |

### 実行中の状態（メモリ）
- `score: Int` / `coins: Int` / `lives: Int (初期3)`
- `scrollSpeed: CGFloat`（難易度で増加、上限あり）
- `phase: Int`（0〜3、距離/時間で進行）
- `isInvincible: Bool`（被弾後の無敵）
- `gameState: enum { title, playing, gameOver }`
