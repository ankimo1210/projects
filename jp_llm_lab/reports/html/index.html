Overview

本レポートは Milestone 1（educational minimum）の実測結果をまとめた静的HTMLです。対象は文字トークナイザ + bigram ベースライン + 約110万パラメータの Classical GPT（Model S）を青空文庫『こころ』で学習したスモーク実験です。高性能なLLMを作ることが目的ではなく、言語モデル内部の観測可能性を確立することが目的です。

Tokenizer（文字トークナイザ）

語彙 = 特殊トークン6種 + 『こころ』に出現する全文字 2,062 種。decode(encode(x)) == x はテストで保証（語彙内文字のみ）。1文字=1トークンなので系列長=文字数になり、コンテキスト256は約256文字に相当する。

例: 私はその人を常に先生と呼んでいた。 → [1382, 59, 41, 58, 177, 88, 672, 55, 270, 1279, 52, 425, 89, 51, 19, 43, 12]

未知文字（例: 学習コーパスに無い字）は <UNK>=id 1 に落ちる。『走れメロス』を同じトークナイザで符号化すると一部の文字が UNK になる — この被覆率の問題が M2 の BPE の動機。

Bigram baseline

$P(x_{t+1}\mid x_t)$ だけを見る最小の言語モデル。カウント（閉形式のMLE+平滑化）とSGDで学習する neural 版（V×V ロジット表）を同一 val スライスで比較。

What（何を表示しているか）
neural bigram の検証損失の推移と、count モデル・一様分布の参照線。
Why（なぜ測定するか）
「ニューラルだから賢い」わけではないことの実証。同じ表現力なら SGD は閉形式解に収束するだけ。
How to read（どう読むか）
オレンジの曲線が灰色破線（count）に近づけば、両者が同じ解に達した証拠。
Observation（観察）
neural val 3.541 vs count val 3.592（α=0.05）。差 0.051 nats で neural がわずかに良い。一様分布 ln V = 7.63 からは両者とも大幅に改善。
Interpretation（解釈）
neural 版はゼロ初期化＝一様事前から出発し、未観測ペアに一様に近いロジットを残す。これが加算平滑化と似た働きをし、α の手動調整なしで同等以上になった。
Caveat（断定できないこと）
差 0.05 nats は平滑化方式の違いであり「ニューラルの表現力」ではない。また val スライス（先頭8,192トークン）での比較。
Next（次に試すこと）
Transformer（下記）が bigram を超える分こそが「文脈」の寄与。M2 でコーパスを変えて再確認。
What（何を表示しているか）
count bigram の加算平滑化 α を振ったときの検証損失。
Why（なぜ測定するか）
平滑化は「未観測ペアにどれだけ確率を残すか」の事前分布。語彙が大きい日本語では α·V が巨大になり効きすぎる。
How to read（どう読むか）
横軸は log スケール。谷が最適な α。
Observation（観察）
α=0.5 では val 4.063 と大きく悪化（α·V ≈ 1034 擬似カウントが実カウントを圧倒）。最適は α≈0.01–0.05。
Interpretation（解釈）
ハイパーパラメータの選択がモデル本体と同程度に損失を動かす好例。
Caveat（断定できないこと）
最適 α はコーパスサイズ・語彙サイズに依存し、この値は『こころ』146K トークン固有。
Next（次に試すこと）
Kneser-Ney など高度な平滑化との比較（教材としては任意）。

Attention: explicit vs SDPA

What（何を表示しているか）
同一パラメータの attention モジュールで、explicit 実装と F.scaled_dot_product_attention の forward+backward 中央値時間。
Why（なぜ測定するか）
教育用の explicit 実装と本番用 fused kernel の出力一致（テスト済: atol 1e-5）と速度差を分離して理解するため。
How to read（どう読むか）
棒が低いほど速い。同じ dtype/T 内で紫（explicit）と緑（SDPA)を比較。
Observation（観察）
SDPA は全条件で高速。T=512・bf16 で最大 6.6×（7.45ms → 1.13ms）。T が伸びるほど差が拡大。
Interpretation（解釈）
explicit 実装は [B,H,T,T] の score 行列をメモリに実体化するため T² のメモリ帯域がボトルネック。SDPA (FlashAttention系) はタイル化で実体化を回避する。
Caveat（断定できないこと）
この規模（d=128）では絶対時間はどちらも小さく、学習全体では data/eval のオーバーヘッドが支配的。速度差 = 精度差ではない（出力は一致）。
Next（次に試すこと）
M3 §14.1 のハードウェア校正で torch.compile / 勾配チェックポイントも含めて総合計測。

What（何を表示しているか）
同一文に対する head 別 attention 分布（init vs 学習後）。
Why（なぜ測定するか）
「学習で attention が何を獲得したか」を最も直接的に観察できる断面。
How to read（どう読むか）
各行は合計1の確率分布。縦の筋=特定トークンが広く参照されている。対角線近傍=局所参照。
Observation（観察）
平均エントロピーは init 1.97 → 学習後 1.69 nats に低下（最深層 layer 3 が 1.48 で最も選択的）。直前トークンへの質量は 0.15 → 0.17 と微増にとどまる一方、先頭トークンへの質量はむしろ 0.15 → 0.13 に減少。学習後のヒートマップでは特定の内容文字（「先」「生」「。」など）への縦筋が目立つ。
Interpretation（解釈）
「学習すると直前トークン注視が支配的になる」という事前予想は、この文では部分的にしか成り立たなかった。エントロピー低下は主に特定キー文字への選択的集中で生じており、深い層ほどその傾向が強い。
Caveat（断定できないこと）
attention 重みが高い ≠ そのトークンが因果的に重要（重み≠寄与）。1文・1シードの定性観察であり、head の役割の一般化はできない。直前トークン率などの数値もこの1文に対するもの。M2 で多数の文について定量化し、ablation/patching で因果性を検証する。

Training dynamics

What（何を表示しているか）
学習損失（per-step ミニバッチ＋固定評価バッチ）と検証損失。ボタンで横軸を step / tokens / 時間に切替可能。
Why（なぜ測定するか）
収束の判断と過学習（暗記）の検出は train と val の乖離で行う。
How to read（どう読むか）
初期値は一様分布の理論値 ln V に一致するはず。train（青）と val（赤）の開きが暗記の兆候。
Observation（観察）
init loss 7.66 ≈ ln V = 7.63（初期化の健全性確認）。最終 train 2.62 / val 3.17、ギャップ 0.55 nats。val ppl は 2127 → 23.9。
Interpretation（解釈）
146K トークンのコーパスを約45周しており、ギャップ 0.55 nats は軽度〜中程度の暗記。教材としては「小データ多エポックで何が起きるか」の実例。bigram val 3.54 → Transformer val 3.17 の改善分（約0.37 nats）が『文脈を使う』能力の寄与。
Caveat（断定できないこと）
単一シード・単一コーパス。val は小説の終盤部分（連続分割）なので文体ドリフトの影響を含む。
Next（次に試すこと）
M2 で本物のコーパススナップショット（重複除去つき）と1エポック未満学習に移行し、暗記を分離。
What（何を表示しているか）
学習率スケジュール（線形ウォームアップ→cosine）と、クリップ前の全体勾配ノルム。
Why（なぜ測定するか）
勾配ノルムの爆発/消失は不安定化の最初のシグナル。クリップ発動率も安定性の指標。
How to read（どう読むか）
破線がクリップ閾値。これを超えた step はクリップが発動している。
Observation（観察）
ウォームアップ後、勾配ノルムは概ね 0.3–0.9 で推移。クリップ発動率は 9%（主に初期）。NaN/Inf 検出は全評価で 0 件。
Interpretation（解釈）
lr=3e-3 はこの規模では安定域。発動が序盤に集中するのは、初期化直後の大きな損失勾配による正常な挙動。
Caveat（断定できないこと）
「クリップ0%が理想」ではない。閾値が緩すぎる可能性と区別できない。
Next（次に試すこと）
M3 §14.2 の LR range test で不安定化点を実測し、閾値と lr を体系的に決める。
What（何を表示しているか）
パラメータ群ごとの update-to-weight 比 ‖ΔW‖/‖W‖（10 step ごとに計測）。
Why（なぜ測定するか）
「学習率が実効的に大きすぎる/小さすぎる」を群別に検出できる、lr そのものより情報量の多い指標。
How to read（どう読むか）
経験則では AdamW で ~1e-3 前後が健全（点線）。群間で桁が揃っているかを見る。
Observation（観察）
全群とも 1e-3 の目安線の近傍で推移し、cosine 減衰に伴って徐々に低下。norm 群は他よりやや高め。
Interpretation（解釈）
群間で極端な乖離がなく、単一 lr で全群が妥当に更新されている。norm（1次元パラメータ）はノルムが小さいため比が出やすい。
Caveat（断定できないこと）
経験則の 1e-3 はタスク・スケール依存の目安にすぎない。
Next（次に試すこと）
M3 で lr を振った際にこの比がどうシフトするか、群別 lr の必要性を検討。
What（何を表示しているか）
パラメータ群別の勾配ノルム時系列（log スケール）。
Why（なぜ測定するか）
損失悪化時に「どの層・どの群で異常が起きたか」を遡れるようにする（spec §12）。
How to read（どう読むか）
群間の相対関係と急変（スパイク）を見る。
Observation（観察）
token_emb と mlp の勾配が大きく、pos_emb は小さい。全群が滑らかに減衰し、スパイクなし。
Interpretation（解釈）
埋め込みは全トークンの誤差を直接受けるため勾配が大きいのは自然。pos_emb の勾配が小さいのは位置情報の寄与が相対的に小さいことを示唆。
Caveat（断定できないこと）
ノルムの大小はパラメータ数にも依存するため、群間比較は同一群の時間変化ほど厳密ではない。
Next（次に試すこと）
M3 の RoPE 置換で pos_emb（学習パラメータ）を除いたときの挙動比較。
What（何を表示しているか）
評価時の residual stream RMS（観測点 × 学習ステップ）。
Why（なぜ測定するか）
活性の爆発・消失、特定層だけの異常（outlier channel 等）の早期検出。
How to read（どう読むか）
色が急激に明るくなる行/列は活性爆発の兆候。深さ方向に緩やかに増えるのは残差加算の正常な蓄積。
Observation（観察）
RMS は深い層ほど大きく、学習に伴い全体に単調増加。tok_emb は init_std=0.02 に対応する小さい値から出発。爆発・NaN なし。
Interpretation（解釈）
pre-LN 構成では residual stream のノルムが層とともに成長するのは既知の挙動で、各 branch の書き込みが蓄積している証拠。
Caveat（断定できないこと）
RMS は分布の1統計量にすぎず、外れ値チャネルの検出には kurtosis / absmax（記録済み）の併読が必要。
Next（次に試すこと）
M2 §11: branch 別寄与 r_l = ‖Δh‖/‖h‖ の層別時系列、外れ値チャネル分析。
What（何を表示しているか）
ウィンドウ平均スループット（tokens/sec）。
Why（なぜ測定するか）
学習時間の見積りと、eval/checkpoint のオーバーヘッド可視化。
How to read（どう読むか）
谷は eval・checkpoint・生成サンプリングを含む区間。
Observation（観察）
定常区間の中央値 ≈ 474,694 tokens/sec（BF16・SDPA・batch 64×256）。全体では 6,553,600 tokens / 41.6s ≈ 157,576 tokens/sec。
Interpretation（解釈）
1.1M パラメータのモデルでは GPU は大部分アイドルで、eval/生成のオーバーヘッド比率が大きい。モデルが大きくなるほどこの比率は下がる。
Caveat（断定できないこと）
VRAM 使用量 0.1GB — この計測はモデルサイズのスケーリング予測には使えない（M3 で実測）。
Next（次に試すこと）

Caveat（断定できないこと）
「逐語再現しない＝暗記していない」ではない（train loss は val より 0.55 nats 低い）。反復は greedy 特有で、サンプリングでは緩和される。

Limitations（この実験から言えないこと）

  • 単一シード・単一コーパス: すべての数値は run-to-run 変動を含む。±いくらかは M3 の複数シード実験まで不明。
  • 多エポック学習: 約45周しており、損失改善の一部は暗記。1エポック未満の大規模コーパス学習（M2/M4）とは力学が異なる。
  • 文字トークナイザ固有: 系列長・語彙被覆・エントロピーの値は BPE では変わる。
  • attention 可視化は説明ではない: 重みの大きさは因果的重要性を保証しない。
  • 確率の信頼性（calibration）未評価: top-1 confidence は上昇したが、それが「正しい確信」かは M5 の reliability diagram まで判定できない。
  • 生成品質の評価は定性的: 反復率・distinct-n 等の定量指標は M5 §16 で導入。
