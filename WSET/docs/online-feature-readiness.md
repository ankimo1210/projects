# 任意オンライン機能の有効化条件

R7のiCloudとAI記述添削は技術検証であり、以下を満たすまでは本番機能として有効化しません。既定状態では外部送信を行いません。

## iCloud手動転送

- iCloud containerとentitlementをDevelopment／Productionそれぞれ設定する。
- 利用者が転送ボタンを押した場合だけ、version付きバックアップDTOをアップロードする。
- 端末内データを正本とし、自動上書きや自動削除を行わない。ダウンロード前に日時、サイズ、SHA-256を表示し、利用者が復元を確定する。
- 同名ファイルは一時ファイルへの書き込み後に置換し、失敗時は既存ファイルと端末内データを保持する。
- 複数端末の競合解決、自動同期、削除同期が必要になった場合は、手動転送とは別機能としてschema migrationと競合fixtureを用意する。

## AI記述添削

- アプリへプロバイダーのAPIキーを埋め込まず、運営者が管理するHTTPSバックエンドだけを指定する。
- ReleaseビルドはInfo.plistの`WSETAIReviewBackendEndpoint`だけを送信先として読み、利用者入力やUserDefaultsの開発用URLを参照しない。値が未設定・不正ならAI添削を無効のままにする。
- 送信先とは別に`WSETAIReviewProductionEnabled`を`true`へ設定した場合だけ本番構成を生成する。このフラグはプライバシー、保持・削除、実機試験、セキュリティレビューの全完了後に設定し、未設定時はfail closedとする。
- 開発用URLの入力欄は`DEBUG`だけにコンパイルし、実バックエンド接続前のローカル検証に限定する。
- URLの認証情報、query、fragmentを拒否し、HTTP redirectでは回答を新しい送信先へ再送しない。
- 送信先ごとに、設問、利用者回答、模範解答、採点基準を送ることへ明示同意を得る。学習履歴と端末識別子は送らない。
- rubric採点を主結果とし、AI自由文は補助フィードバックとして扱う。評価fixtureと人手比較が基準未達なら有効化しない。
- 本番バックエンドは保存しない処理を原則とする。障害解析で一時保持する場合でも30日以内の上限、暗号化、アクセス制御、削除方法、問い合わせ先を公開する。
- 同意取消は将来の送信を停止する。既送信データの削除が必要な構成では、削除APIまたは運営者への削除依頼導線を同じ画面へ用意する。
- レート制限、タイムアウト、最大応答1 MB、schema検証、監査ログから回答本文を除外することを実バックエンドで確認する。

## Go/No-Go

本番有効化には、プライバシーポリシーとApp Privacy回答の更新、実機での同意／取消／失敗復帰、保持・削除試験、セキュリティレビューが必要です。1つでも未完了なら設定画面の「技術検証」表示と無効状態を維持します。
