### 3. 周辺施設情報

物件分析ページに「周辺施設」セクションを追加。

現在取得している項目:

- コンビニ
- スーパー
- 駅・バス停
- パチンコ
- 飲食店
- 学校
- 病院・診療所
- 公園

表示内容:

- 1km 以内の件数
- 500m 以内の件数
- 最寄り施設までの距離
- 近い施設の一覧
- 一覧の施設カテゴリフィルター
- カテゴリごとの表示件数調整

実装方針:

- OpenStreetMap Overpass API を利用。
- 複数カテゴリを 1 回の Overpass クエリにまとめ、API レート制限を受けにくくした。
- 取得結果は `st.cache_data(ttl=86400)` で 1 日キャッシュ。
- Overpass に未登録の施設は検出できないため、UI 上でも簡易チェックとして明示。

主な変更:

- `facility_sources.py`
- `tests/test_facility_sources.py`
- `ui/property_tab.py`

### 5. 地形・水害リスク簡易チェック

物件分析ページに「地形・水害リスク（簡易）」セクションを追加。

現在表示している項目:

- 標高
- 標高区分
  - 3m 未満
  - 10m 未満
  - 10m 以上
  - 不明
- 最寄り水辺までの距離
- 1km 内の河川・水辺件数
- 近い河川・水辺の一覧

実装方針:

- 標高は国土地理院の標高取得プログラムを利用。
- 河川・水辺は OpenStreetMap Overpass API を利用。
- 取得結果は `st.cache_data(ttl=86400)` で 1 日キャッシュ。
- これは自治体ハザードマップや浸水想定区域の正式判定ではなく、投資判断のための簡易シグナルとして扱う。

実API疎通確認:

- 新宿付近で標高 `37.4m` を取得。
- 同地点 1km 内で水辺候補 `29件` を取得。

主な変更:

- `terrain_sources.py`
- `tests/test_terrain_sources.py`
- `ui/property_tab.py`

## 現時点の注意点

### 周辺施設

- OpenStreetMap の登録状況に依存する。
- コンビニやスーパーは比較的使いやすいが、学校・病院・公園・パチンコなどは地域により登録密度が変わる。
- 施設の「有無」や「距離」は価格説明変数として使えるが、まずは表示・確認用途に留めるのが安全。

### 河川・水辺

- 現在の距離は OSM 要素の座標または中心点に対する簡易距離。
- 線形の河川ポリラインに対する厳密な最近傍距離ではない。
- 投資判断の警戒シグナルには使えるが、浸水リスクそのものの判定ではない。

### 標高

- 国土地理院APIの `hsrc` により精度が異なる。
- 低地判定の閾値は暫定。
- 海抜が高くても内水氾濫・土砂災害・液状化リスクは別に存在する。

## 次のエンハンスメントプラン

### A. 施設情報の特徴量化

目的:

- 表示だけでなく、価格説明変数や投資判断指標として使える形にする。

追加候補:

- カテゴリ別の最寄り距離
- カテゴリ別の 300m / 500m / 1km 件数
- 最寄駅・バス停の距離
- コンビニ・スーパーの生活利便スコア
- 学校・公園・病院のファミリー需要スコア
- パチンコ・風俗・大規模騒音源などのネガティブ施設スコア

実装方針:

1. 物件分析ページでは現状のリアルタイム取得を継続。
2. 将来的に `property_features` のようなテーブルを作り、座標ごとの施設特徴量を保存。
3. 近傍公示地価・取引価格にも同じ特徴量を付与し、価格差の説明に使う。

### B. 施設カテゴリの拡張

プラス要因:

- 駅
- バス停
- スーパー
- ドラッグストア
- コンビニ
- 小中学校
- 大学
- 病院・診療所
- 公園
- 図書館
- 役所・行政窓口
- 商業施設

マイナス要因:

- パチンコ
- 風俗関連
- 産業施設
- ごみ処理施設
- 火葬場・墓地
- 高速道路・幹線道路近接
- 鉄道線路近接
- 騒音源になりやすい大型施設

優先順位:

1. ドラッグストア
2. 商業施設
3. 風俗関連
4. 幹線道路・鉄道近接
5. ごみ処理施設などの嫌悪施設

### C. 公式ハザードデータ連携

目的:

- 現在の「簡易シグナル」から、より正式なリスク判定に近づける。

候補データ:

- 洪水浸水想定区域
- 津波浸水想定
- 高潮浸水想定
- 土砂災害警戒区域
- 液状化リスク
- 地震動予測
- 地盤分類

実装方針:

1. まず都道府県または対象エリアを限定して、国土数値情報などの公的GISデータをローカルに取り込む。
2. Shapely で物件座標とポリゴンを空間判定する。
3. 判定結果を物件分析ページに表示する。
4. バッチ処理で公示地価・取引価格地点にも同じリスク属性を付与する。

初期実装候補:

- 洪水浸水想定区域に含まれるか
- 想定浸水深ランク
- 土砂災害警戒区域に含まれるか
- 海岸線・主要河川からの距離

### D. 地形情報の高度化

現在:

- 単地点の標高
- OSM ベースの水辺距離

追加候補:

- 周辺 100m / 250m / 500m の標高差
- 低地・谷地・台地の簡易分類
- 傾斜度
- 河川までの厳密な最近傍距離
- 海岸線までの距離
- 標高 3m 未満エリアへの近接

実装方針:

1. 国土地理院の標高タイルを使い、周辺グリッドの標高を取得。
2. 中心点との差分、最低標高、標高レンジ、傾斜の代理変数を作る。
3. 投資判断では「低地」「周辺より低い」「川に近い」を別々のシグナルとして扱う。

### E. UI改善

候補:

- 周辺施設と地形リスクをタブ化する。
- 重要指標だけを上部にカード表示し、詳細は expander に入れる。
- 施設一覧に距離順・カテゴリ順の切り替えを追加する。
- 地形・水害セクションに「注意」「中立」「良好」の簡易ラベルを追加する。
- 地図上に周辺施設・水辺候補を表示する。

優先順位:

1. 地図上の施設・水辺プロット
2. 周辺施設一覧の距離順ソート
3. 簡易リスクラベル
4. 詳細情報のタブ化

### F. データ永続化・再利用

目的:

- 同じ座標や近い座標で外部APIを何度も叩かない。
- 将来の価格モデル・ランキング・比較分析に使えるようにする。

候補テーブル:

- `property_location_features`
- `nearby_facility_features`
- `terrain_features`
- `hazard_features`

保存する値:

- 緯度経度を丸めたキー
- 取得日時
- 半径
- カテゴリ別件数
- カテゴリ別最寄り距離
- 標高
- 水辺距離
- ハザード判定
- データソースとバージョン

### G. 価格説明モデルへの接続

目的:

- 施設・地形・ハザードが価格にどう効くかを定量的に見る。

初期アプローチ:

- 公示地価・取引価格地点に特徴量を付与。
- 価格単価を目的変数にする。
- 市区町村、用途、駅距離、面積、年、施設特徴量、地形特徴量を説明変数にする。
- まずは線形回帰・ランダムフォレスト・LightGBM などで比較。

表示候補:

- 入力物件の価格が、周辺地価・施設利便性・地形リスクから見て高いか安いか。
- プラス要因とマイナス要因の寄与。
- 同条件に近い比較対象地点の一覧。
