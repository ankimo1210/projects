## 0. エグゼクティブサマリー

本アプリは、個人投資家が不動産会社・仲介会社・販売会社から提供される物件情報や資料をそのまま信じず、購入前に自分で収支・融資・税金・修繕・出口・代替資産比較を検証するための **買付前監査ツール** である。

中心思想は以下。

> 不動産会社にユーザー情報を売らない。  
> 営業リード化しない。  
> 個人投資家が自分で判断するための中立分析ツールを作る。

既存の不動産投資サービスは、物件掲載、投資家マッチング、査定、営業送客、保有物件管理に寄っているものが多い。一方、本アプリは「物件を売る側」ではなく「買う側」のために、業者資料・レントロール・販売図面・URL・公的データを読み込み、以下を提示する。

- 資料が足りているか
- どの前提が甘いか
- 税前/税後キャッシュフローはどうか
- 金利上昇・空室・修繕・出口悪化に耐えられるか
- 最大いくらまでなら検討可能か
- そもそも不動産ではなく株式・債券・定期預金等の方が合理的ではないか
- ユーザー自身が不動産投資に向いているか

最初のMVPでは、区分マンション・一棟アパートを対象に、URL貼り付けと資料アップロードを入口とする。手入力は最小化し、AIで抽出した項目をユーザーが確認・修正するUXを目指す。

## 1. プロダクトの立ち位置

### 1.1 コアコンセプト

**個人投資家のための不動産買付前監査OS**

ユーザーが見つけた物件・業者から受け取った資料を、以下の観点から監査する。

1. 収支
2. 融資
3. 税金
4. 修繕・CAPEX
5. 売却・出口
6. 賃料・価格妥当性
7. 資料充足率
8. 前提の甘さ
9. クロスアセット比較
10. ユーザー本人の投資適性

### 1.2 ブランド上の約束

LP、FAQ、利用規約、アプリ内に明記すべき方針。

```text
当サービスは、不動産会社・仲介会社・販売会社にユーザー情報を販売しません。

入力された物件情報・資産情報・分析履歴は、ユーザー本人の分析体験改善と匿名統計の作成に利用します。

個別の明示同意なしに、不動産会社・金融機関・営業会社へ情報提供することはありません。

物件提案・営業電話・面談誘導を目的としたサービスではありません。
```

### 1.3 競合との差別化

| 競合タイプ | 主な価値 | 弱点/ユーザー不安 | 本アプリの差別化 |
|---|---|---|---|
| 不動産ポータル | 物件掲載、検索 | 売り手側の情報中心 | 掲載情報を監査する |
| 不動産投資アプリ | 収支管理、物件提案 | 個人情報・保有情報の送客懸念 | 業者に売らない |
| 収支シミュレーター | CF/IRR計算 | 手入力が多い、資料監査が弱い | PDF/レントロール解析 |
| AI査定/賃料査定 | 価格・賃料推定 | B2B/業者向けが多い | 個人投資家向けDD |
| 金融資産バックテスト | 株/債券比較 | 実物不動産個別案件と比較できない | 実物不動産 vs 他資産比較 |

## 2. 事業方針

### 2.1 開始形態

当面は法人化せず、個人事業主として開始する。

理由:

- 初期赤字100万円程度を許容
- 給与所得が大きく、初期損失をなるべく損出ししたい
- 法人化は利益が安定してからでよい
- まずMVP検証が優先

### 2.2 税務運用方針

最初から以下を整備する。

| 項目 | 方針 |
|---|---|
| 開業届 | 出す |
| 青色申告承認申請 | 出す |
| 会計 | freee / マネーフォワード / 弥生など |
| 口座 | 事業用に分ける |
| クレカ | 事業用に分ける |
| 帳簿 | 複式簿記 |
| 証憑 | AI利用料、サーバー費、ドメイン、法務、デザイン等を保存 |
| 事業実態 | LP、利用規約、GitHub、βユーザー、課金導線を残す |
| 法人化 | 月商・利益が見えてから検討 |

注意点:

- 個人事業の赤字が給与所得と損益通算できるかは、事業所得として認められる実態が重要。
- 雑所得扱いされると赤字の損益通算は困難。
- 税務判断は最終的に税理士確認が必要。

### 2.3 収益モデル

不動産会社への送客収益には依存しない。

| 収益源 | 位置づけ |
|---|---|
| Free広告 | 集客・無料提供の補助 |
| Light課金 | 広告なし、保存件数増加 |
| Pro課金 | 詳細分析、比較、PDF、最大買付価格 |
| 単発レポート課金 | サブスクが嫌なユーザー向け |
| 投げ銭 | 中立運営への共感課金 |
| Strong Supporter | beta機能先行アクセス、開発支援 |
| 匿名統計レポート | 将来のデータ商品 |
| B2B版 | 将来。個人投資家向けの思想を壊さない範囲 |

| プラン | 価格 | 内容 |

### 2.5 投げ銭・サポーター設計

役割を明確に分ける。

| 種別 | 目的 |
|---|---|
| 投げ銭 | 気軽な単発応援 |
| Strong Supporter | 継続支援 + beta機能先行利用 |
| Pro | 実用機能の利用 |

注意点:

- 「寄付」ではなく「応援課金」「サポート」と呼ぶ。
- beta機能を提供する場合、実質的には有料サービスなので特商法表示・返金方針・提供内容を明記する。
- ポイント制や前払式支払手段に近い設計は避ける。

## 3. 対象ユーザー

### 3.1 初期ターゲット

- 日本国内の個人不動産投資家
- 区分マンション・一棟アパートを検討している人
- 業者資料やポータル情報をそのまま信じたくない人
- 表面利回りではなく税後CFや出口まで見たい人
- 本業があり、営業面談に巻き込まれず自分で判断したい人
- 投資リテラシーは中級程度だが、不動産DDの型を持ちたい人

### 3.2 ユーザータイプ

| タイプ | 特徴 | 提供価値 |
|---|---|---|
| 初心者寄り | 表面利回りに引かれやすい | リテラシーテスト、注意喚起、学習導線 |
| 中級者 | 物件比較をしている | 資料監査、最大買付価格、比較ボード |
| 高所得者 | 節税・減価償却に関心 | 税後CF、デッドクロス、売却税 |
| 忙しい投資家 | 手入力したくない | PDF/レントロールAI解析 |
| ETF向きの人 | 不動産に向かない可能性 | クロスアセット比較 |
| 強いユーザー | beta機能に協力 | Strong Supporter |

### 6.8 業者資料の甘さチェック

| チェック | 内容 |
|---|---|
| 表面利回りのみ強調 | 経費控除後のNOI利回りを再計算 |
| 満室想定のみ | 現況賃料・空室・滞納を分離 |
| 固定資産税未記載 | 税後CFが過大に見える可能性 |
| 修繕費未記載 | 築古で危険 |
| 管理費低すぎ | 実態確認が必要 |
| 敷金/原状回復未考慮 | 退去時CFがぶれる |
| 出口前提なし | IRRが見えない |
| 減価償却前提なし | 税後CFが見えない |
| レントロール更新日不明 | 賃料情報の鮮度が低い |
| 滞納/フリーレント不明 | 実効賃料が不明 |

## 7. 収支・税務・融資分析エンジン

### 7.1 基本計算

| 指標 | 内容 |
|---|---|
| GPI | 満室想定総収入 |
| Vacancy Loss | 空室損 |

## 8. スコアリング設計

### 8.1 総合スコア

100点満点。買い推奨ではなく、分析上の健全性・リスク耐性を示す。

| 項目 | 配点 |
|---|---:|
| 価格妥当性 | 20 |
| 賃料妥当性 | 15 |
| 税前/税後CF | 20 |
| 融資耐性 | 15 |
| 出口耐性 | 15 |
| 修繕・CAPEX耐性 | 10 |
| データ信頼度 | 5 |

表示例:

```text
総合スコア: 68 / 100
評価: 中立〜やや割高

サマリー:
1. 表面利回りは悪くないが、管理費・修繕積立金込みのNOI利回りは弱い。
2. 金利+1.0%時に税後CFが赤字化しやすい。
3. 出口利回りを保守的に置くと、10年IRRが大きく低下する。
```

### 8.2 データ信頼度スコア

総合スコアとは別に表示する。

| 項目 | 内容 |
|---|---|
| 資料充足率 | 必要資料が揃っているか |
| 抽出信頼度 | AI抽出の確信度 |
| 手修正有無 | ユーザー確認済みか |
| 出典鮮度 | レントロール更新日等 |
| 矛盾有無 | 販売図面とレントロールの整合性 |

例:

```text
分析信頼度: 62 / 100
簡易分析は可能ですが、固定資産税・修繕履歴・滞納有無が不明なため、詳細判断には追加確認が必要です。
```

### 8.3 最大買付価格

ユーザーの投資基準から逆算。

入力:

- 目標IRR
- 最低DSCR
- 最低年間CF
- 自己資金上限
- 金利
- 融資期間
- 売却Cap Rate
- 保有期間

出力:

```text
現在価格: 3,980万円
最大許容価格: 3,420万円
安全価格: 3,200万円
必要指値: -560万円

制約要因:
- DSCR 1.2倍条件が最も厳しい
- 金利+1.0%時にCFが赤字化
- 売却Cap Rate+0.5%でIRRが目標未達
```

### 8.4 感応度分析

最低限のシナリオ。

| シナリオ | ショック |
|---|---|
| 賃料下落 | -5%、-10% |
| 空室率悪化 | +5pt、+10pt |
| 金利上昇 | +0.5pt、+1.0pt |
| 修繕費増加 | 1.5倍、2.0倍 |
| 売却Cap Rate悪化 | +0.25pt、+0.5pt |
| 売却価格下落 | -5%、-10% |
| 複合ストレス | 上記を組み合わせ |

## 9. クロスアセット比較機能

### 9.1 目的

不動産単体ではなく、同じ自己資金を以下に投資した場合と比較する。

- 実物不動産
- 日本株インデックス
- 米国株インデックス
- 全世界株インデックス
- J-REIT
- 国内債券
- 米国債
- 円定期預金
- MMF/短期金利商品

### 9.2 コア問い

```text
この自己資金を本当にこの不動産に入れるべきか？
それとも株式・債券・預金・REITの方が合理的か？
```

### 9.3 比較指標

| 指標 | 不動産 | 他資産 |
|---|---|---|
| 期待リターン | 税後IRR | CAGR/期待リターン |
| インカム | 税後CF | 配当/利息 |
| 下落リスク | 売却Cap Rate/価格下落 | ボラ・最大DD |
| 流動性 | 低い | 高い |
| 分散性 | 低い | 高い |
| 手間 | 高い | 低い |
| 借入リスク | あり | 原則なし |
| 為替リスク | 原則なし | 米国資産はあり |
| 税務複雑性 | 高い | 低〜中 |

### 9.4 不動産リスクプレミアム

```text
不動産リスクプレミアム = 物件の税後IRR - 代替資産期待リターン
```

判定例:

| プレミアム | 判定 |
|---:|---|
| +5%以上 | リスクを取る余地あり |
| +2〜5% | 条件次第 |
| 0〜2% | 微妙 |
| マイナス | 他資産の方が合理的な可能性 |

### 9.5 表示例

```text
自己資金500万円の使い道比較

A. この一棟アパート
- 税後IRR: 4.1%
- 年間税後CF: +8万円
- ストレス時IRR: -1.5%
- 流動性: 低
- 手間: 高
- 集中リスク: 高

B. 全世界株インデックス
- 期待年率: 5.0%
- 流動性: 高
- 手間: 低
- 分散: 高

C. 国内債券/定期預金
- 期待年率: 0.5〜1.0%
- 流動性: 高
- 元本変動: 低〜中

参考判定:
この物件は、株式インデックスと比べて期待リターンが高いとは言いにくく、借入・修繕・流動性リスクに見合うプレミアムが不足しています。
```

### 9.6 金融商品取引法への注意

クロスアセット比較は投資助言リスクが上がるため、以下を守る。

避ける表現:

```text
このETFを買うべき
この物件ではなくVTIを買いましょう
今は債券を買うタイミングです
あなたには株70%、債券30%が最適です
```

使う表現:

```text
資産クラス間の参考シナリオ比較
過去データ・仮定に基づく試算
特定金融商品の売買推奨ではありません
最終的な投資判断はユーザー自身で行ってください
```

初期は個別銘柄ではなく、資産クラス・指数レベルに留める。

| 最大買付価格計算 | 最適化問題として解く |

↓
信頼度スコア
↓
計算エンジンへ渡す

結果サマリー生成

いきなり独自モデルを作らず、以下で進める。

1. OpenAI API + 専用スキーマ

3. 項目別正解率を評価

## 13. 法務・規約・リスク設計

### 13.1 不動産投資助言・宅建業リスク

避ける表現:

```text
買うべき
おすすめ
儲かる
この物件は狙い目

見送り推奨
```

使う表現:

```text

前提条件付き
割安傾向
割高傾向
要追加確認
税後CFが弱い
DSCR耐性が弱い
出口前提への依存が高い
```

### 13.2 金融商品取引法リスク

クロスアセット比較では、金融商品の個別推奨を避ける。

- 個別銘柄/ETF/債券の推奨はしない
- 資産クラス/指数レベルの参考比較に留める
- 有料で「あなたはこのETFを買うべき」とは言わない
- 免責を明記する
- 法務レビュー必須

### 13.3 スクレイピングリスク

方針:

- URL解析はユーザーが貼ったURLの入力補助に留める
- 一覧クロールや大規模収集は後回し
- allowlist方式
- サイト別kill switch
- 対象サイトから警告が来たら該当サイト停止
- 手入力/資料アップロードで成立する設計にする

### 13.4 個人情報・プライバシー

方針:

- 電話番号・勤務先・実名を初期必須にしない
- 保有物件情報の入力は任意
- ユーザー資料を業者に渡さない
- PIIは最小限
- 削除機能を用意
- 外部AI API利用を明記
- 匿名統計利用を明記

### 13.5 特商法表示

有料機能、投げ銭、Strong Supporter、単発レポート課金を行う場合、以下を整備する。

- 事業者名
- 住所/連絡先
- 販売価格
- 支払方法
- 提供時期
- 返金方針
- 解約方法
- beta機能の不確実性
- 免責

### 13.6 返金方針

案:

```text
初回課金から7日以内で、重大不具合またはサービス未利用の場合は返金可。
分析済み件数が多い場合、または単発レポート生成後は原則返金対象外。
beta機能は予告なく変更・停止される場合があります。
```

## 19. 初期プロンプト/AIスキーマ方針

### 19.1 資料分類プロンプト

```text
あなたは日本の不動産投資資料を分類するアシスタントです。
アップロードされた資料が以下のどれに該当するか判定してください。

- property_brochure
- rent_roll
- income_statement
- fixed_asset_tax_statement
- registry_certificate
- repair_history
- management_report
- lease_contract
- unknown

不明な場合は unknown とし、推測で断定しないでください。
出力は指定JSONスキーマに従ってください。
```

### 19.2 抽出方針

- 不明な項目は null
- 推測値は inferred=true
- confidenceを項目ごとに付与
- 賃料の月額/年額を区別
- 円/万円/千円を正規化
- 日付をISO形式に正規化
- 面積単位を㎡に正規化
- 税込/税別が不明なら notes に記録

## 21. 主要リスクと対策

| リスク | 内容 | 対策 |
|---|---|---|
| 税務 | 事業所得認定されない | 事業実態・帳簿・課金導線・税理士確認 |
| スクレイピング | 規約違反・警告 | URL補助、kill switch、手入力/資料で代替 |
| 個人情報 | 資料内PII | マスク、最小保存、削除可能 |
| AI誤抽出 | 数値ミス | 信頼度、ユーザー確認、validation |
| 計算ミス | CF/税/IRR誤り | 自動テスト、計算ロジック固定 |
| 投資助言 | 買うべき表現 | 参考試算、資産クラス比較、法務レビュー |
| 宅建業っぽさ | 仲介・送客に見える | 物件紹介しない、業者に渡さない |
| 広告不信 | 中立性低下 | 広告主とスコア分離、明示 |
| 収益不足 | 送客しないため低収益 | Pro、単発課金、Supporter、レポート |
| LLMコスト | PDF解析が高い | 軽量モデル分類、高性能モデル再解析、キャッシュ |
| betaクレーム | 不安定機能 | beta明示、変更/停止可能性、返金方針 |

## 22. LPコピー案

### 案1

```text
不動産会社に営業されない、個人投資家のための物件分析ツール。

物件URLや販売資料を入れるだけで、収支・融資・税金・出口リスクをチェック。
営業リード化なし。個人情報の不動産会社提供なし。
```

### 案2

```text
買う前に、自分で検証する。

表面利回りだけでは見えない、税後CF・金利耐性・出口リスクを可視化。
不動産会社に情報を渡さない、中立の買付前監査ツール。
```

### 案3

```text
その物件、本当に割安ですか？

営業されずに、投資家自身で収支・融資・修繕・売却リスクを確認。
物件を売る側ではなく、買う側のための分析アプリ。
```

### 案4

```text
業者資料を、そのまま信じない。

販売図面・レントロール・収支表をAIで読み取り、
足りない資料、甘い前提、確認すべき質問を見える化します。
```

### 案5

```text
その自己資金、本当に不動産に入れるべきですか？

実物不動産、株式、債券、J-REIT、定期預金を同じ土俵で比較。
買わない選択肢も含めて、個人投資家の判断を支援します。
```

## 23. 最終コンセプト

本アプリは、単なる「不動産投資シミュレーター」ではない。

最終的には以下を目指す。

> 個人投資家のためのクロスアセット投資判断OS。  
> 実物不動産の資料をAIで読み取り、収支・融資・税金・修繕・出口を監査し、  
> 株式・債券・定期預金などの代替資産と比較しながら、  
> ユーザーが自分の頭で判断できる材料を提供する。

最重要の差別化は以下。

1. 業者に売らない
2. 営業リード化しない
3. 買わない選択肢も出す
4. 手入力ではなく資料解析を中心にする
5. 収支だけでなく資料不足と前提の甘さを監査する
6. 不動産単体ではなく他資産と比較する
7. ユーザー本人が不動産投資に向いているかも判定する
