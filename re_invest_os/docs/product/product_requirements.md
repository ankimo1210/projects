## 1. プロダクトゴール

個人投資家が、業者から渡された物件情報・資料をそのまま信じず、購入前に自分で監査するための買付前DDツール。

## 2. ターゲットユーザー

| タイプ | 主シナリオ |
|---|---|
| 初心者寄り | リテラシーテスト → 学習導線 → 簡易分析 |
| 中級者 | URL貼り付け+資料アップロード → 比較ボード → 最大買付価格 |
| 高所得層 | 税後CF・デッドクロス・売却税の精緻計算 |
| 忙しい投資家 | PDF/レントロールAI抽出を最大活用 |
| ETF向きの人 | クロスアセット比較で「買わない」を選ぶ |
| Strong Supporter | beta機能先行・開発投票 |

詳細は `blueprint_v0_1.md` 3章。

## 8. 法務・表現 (詳細は legal_risk_checklist.md)

### 避ける表現

```
買うべき / おすすめ / 儲かる / 狙い目 / 購入推奨 / 見送り推奨
このETFを買うべき / VTIを買いましょう / 株70%債券30%が最適
```

### 使う表現

```
参考試算 / 前提条件付き / 割安傾向 / 割高傾向 / 要追加確認
税後CFが弱い / DSCR耐性が弱い / 出口前提への依存が高い
資産クラス間の参考シナリオ比較
特定金融商品の売買推奨ではありません
```

## 9. データ最小化・PII方針

- 電話番号・勤務先・実名を必須にしない
- 保有物件情報の入力は任意
- 資料内の賃借人名・電話番号・メールはLLM送信前にマスク
- ユーザー削除APIで全データを論理削除 → 30日後に物理削除
- 外部AI API利用と業者非提供を利用規約・プライバシーポリシーに明記

## 10. MVPでやらないこと (青写真20章)

- 不動産会社への送客
- 一括問い合わせ
- 電話番号必須登録
- 業者からのオファー
- 物件紹介・マッチング
- 大規模スクレイピング
- 個別金融商品の売買推奨
- 実質的な投資助言
- 銀行提出用の正式資料保証
- 税務・法務判断の断定
- AIによる最終投資判断
- ポイント制課金 (前払式支払手段リスク回避)
- 完全自動の買う/買わない判定

## 11. 受け入れ基準 (MVP出荷条件)

1. 楽待・SUUMO URLを5サンプル投入し、抽出成功率80%以上
2. 販売図面PDF 10サンプルで主要項目 (価格・所在地・面積・賃料・築年月) 抽出成功率80%以上
3. レントロールPDF/Excel 5サンプルで部屋単位の賃料が80%以上抽出
4. CF/IRR/DSCR/最大買付価格のユニットテストが全パス
5. Stripe Free→Pro アップグレード/解約のE2Eテスト通過
6. 利用規約・プライバシー・特商法・免責の各ページが公開
7. 業者送客しない宣言がLP・規約・アプリ内3箇所以上で確認できる
8. Sentry連携でエラー監視
9. PostHogで主要KPIイベント計測
10. ユーザー削除APIが動作

## 12. オープン論点 (Phase 0で決める)

- ホスティング: Fly.io か Cloud Run か (api)
- LLM選定: GPT-4.1 / Claude Sonnet 4.6 / Claude Haiku の役割分担
- レントロール抽出: Vision モデル直接 vs OCR→LLM
- ユーザー入力の融資条件テンプレ: 銀行別 or 汎用
- 比較ボードのデータモデル: 永続 or セッション
- リテラシーテスト: 設問プール (10章) を誰が監修するか
