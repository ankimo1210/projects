# Legal Risk Checklist — re_invest_os

作成日: 2026-05-12
ベース: blueprint_v0_1.md §13
**重要**: 本ドキュメントは Claude が青写真ベースで作成した一次草案であり、**正式運用には弁護士・税理士による専門家レビューが必須**。

---

## 0. 全体方針

re_invest_os は **個人投資家のための買付前監査ツール** であり、以下を **目的としない**:

- 不動産仲介・媒介 (宅建業)
- 投資助言 (投資助言業)
- 金融商品の販売・推奨 (金融商品取引法)
- 不動産の購入推奨 / 売却推奨

サービス全体・LP・利用規約・アプリ内表示で、**「参考試算ツール」「中立な検算ツール」** であることを一貫して表現する。

---

## 1. 不動産投資助言・宅建業リスク

### 1.1 法律根拠
- 宅地建物取引業法
- 金融商品取引法 (投資助言・代理業)
- 景品表示法 (誇大表示)
- 特定商取引法

### 1.2 避ける表現 (NG)

| カテゴリ | NG例 |
|---|---|
| 購入推奨 | 「買うべき」「おすすめ」「狙い目」「絶好の物件」 |
| 売却推奨 | 「今が売り時」「保有を継続すべき」 |
| 利益断定 | 「儲かる」「必ず利益が出る」「リスクなし」 |
| 価格断定 | 「割安です」「適正価格」「安すぎる」 |
| 助言断定 | 「あなたは買うべきです」「やめておくべき」 |
| 営業 | 「専門家に相談を」 (我々が紹介する場合) |
| 個別銘柄推奨 | 「VTIを買うべき」「このETFが最適」 |

### 1.3 使う表現 (OK)

| カテゴリ | OK例 |
|---|---|
| 中立試算 | 「参考試算」「前提条件付き」「シナリオ分析」 |
| 傾向描写 | 「割安傾向」「割高傾向」「市場平均より高い/低い」 |
| 検証推奨 | 「要追加確認」「資料不足」「前提の確認をおすすめします」 |
| ファクト描写 | 「税後CFが弱い」「DSCRが基準を下回る」「出口前提への依存が高い」 |
| 主体回避 | 「あなたの基準では…」「指定した条件下では…」 |
| 免責 | 「最終的な投資判断はユーザー自身の責任で」 |

### 1.4 自動チェック (推奨実装)

- LLM出力の post-processing で NG ワードを regex 検出 → 検出時は再生成
- LP・規約・アプリ内表示の文字列を string-search でCIチェック (`grep -r "買うべき" docs/ apps/`)

## 2. 金融商品取引法リスク (クロスアセット比較)

### 2.1 リスクの中心

クロスアセット比較機能で、**個別金融商品の推奨をしない**。

### 2.2 NG / OK

| NG | OK |
|---|---|
| 「VTIを買いましょう」 | 「米国株インデックスの参考リターン」 |
| 「今は債券を買うタイミング」 | 「国内債券の参考利回り」 |
| 「あなたには株70%・債券30%が最適」 | 「資産クラスごとの参考シナリオ比較」 |
| 「不動産より株式の方が良い」 | 「不動産税後IRR vs 各資産クラス参考リターンの差」 |
| 個別ETF・銘柄名の言及 | 資産クラス・指数レベルでの参考表示 |

### 2.3 実装上のガード

- ベンチマークデータは **資産クラス単位のみ** (`docs/architecture/calculation_engine_spec.md §5.9`)
- 個別銘柄テーブルを持たない
- "VT / VTI / TOPIX" の固有名は note 欄に控えめに記載するが、推奨表現は禁止
- 結果画面に常に免責文を表示 (cross_asset.py の `CrossAssetResult.disclaimer` を表示)

## 3. スクレイピング・知的財産

### 3.1 方針

- **ユーザーが貼ったURL** の入力補助に限定
- 一覧クロール・大規模収集は行わない
- allowlist 方式 (起動時: rakumachi / suumo / kenbiya)
- 各サイトの利用規約を遵守 (robots.txt含む)

### 3.2 サイト別 kill switch

- 環境変数 `SCRAPING_DISABLED_HOSTS` で即停止
- 警告・停止要請があった場合は即座に該当ホストを無効化
- 代替: 手動入力フォーム / 資料アップロードで分析継続可能な設計

### 3.3 表示の扱い

- 取得した HTML / 画像はユーザー専用 (他ユーザーには非表示)
- スクリーンショットや画像の二次配布なし
- 物件情報の引用は最小限、原典URLへのリンクを必ず併記

## 4. 個人情報・プライバシー

### 4.1 PII 取り扱い

| 項目 | 方針 |
|---|---|
| 実名 | 必須にしない (display_name のみ任意) |
| 電話番号 | **取得しない** |
| 住所 | **取得しない** (物件住所は除く) |
| 勤務先 | 取得しない |
| 銀行口座 | 取得しない (Stripe経由のみ) |
| 保有物件情報 | 任意入力 |
| アップロード資料内のPII | LLM送信前にマスク (氏名・電話・メール) |
| 賃借人個人名 | レントロール抽出時にマスク |

### 4.2 LLM への送信

- 利用規約・プライバシーポリシーに「外部AI API利用」を明記
- マスク処理した上で送信
- LLM 提供者 (Anthropic / OpenAI) との DPA を確認
- 国境を越えるデータ移転を明示

### 4.3 第三者提供

```
不動産会社・仲介会社・販売会社へのユーザー情報・資料提供は一切行わない。
これはサービスの中核思想として技術的にも担保する:
- 業者向け API を一切実装しない
- 外部送信ログを監査ログ化し、月次レビュー
- データ販売契約を一切結ばない
```

### 4.4 削除権

- ユーザー削除API → 論理削除 → 30日後物理削除
- 退会後も統計目的の **匿名化済み集計データ** は保持する場合がある (利用規約に明記)
- 個別の削除要求 (GDPR Article 17 相当) には30日以内に対応

## 5. 特定商取引法表示

有料機能・投げ銭・Supporter課金・単発レポートを実施する場合、以下を整備:

### 5.1 必須記載 (特商法11条)

| 項目 | 内容 |
|---|---|
| 事業者名 | (個人事業) Kazumasa (姓名フル) |
| 所在地 | 住所 (請求があれば遅滞なく開示できる旨表記でOK) |
| 連絡先 | メールアドレス (電話番号は任意) |
| 販売価格 | 各プラン明示 (税込) |
| 商品の引渡時期 | 「決済完了後即時利用可能」 |
| 支払方法 | クレジットカード (Stripe経由) |
| 支払時期 | 月次自動課金 / 単発即時 |
| 返品・解約 | 後述 §5.3 |
| 動作環境 | ブラウザ要件 |

### 5.2 課金プラン別の追加注意

| プラン | 注意 |
|---|---|
| 単発レポート | 提供後のキャンセル不可を明記 |
| サブスク | 解約方法・解約タイミング・日割りの有無を明記 |
| 投げ銭 | 「寄付」ではなく「応援課金」「サポート」と呼ぶ。ポイント制・前払式に該当しないよう設計 |
| Strong Supporter | beta機能の不確実性・変更・停止の可能性を明記 |

### 5.3 返金方針

```
初回課金から7日以内で、重大不具合またはサービス未利用の場合は返金可。
分析済み件数が多い場合、または単発レポート生成後は原則返金対象外。
beta機能は予告なく変更・停止される場合があります。
Strong Supporter は1か月分を最小単位とし、月途中解約での日割り返金はしません。
```

## 6. 利用規約 (草案要点)

### 6.1 サービスの性質

```
本サービスは、ユーザーが入力・アップロードした情報および公開情報に基づき、
不動産投資に関する参考試算・シミュレーションを提供するものです。
特定の不動産・金融商品の購入・売却を推奨するものではなく、投資助言業・宅建業に
該当する行為を行うものではありません。
```

### 6.2 ユーザーの責任

```
本サービスが提供する情報は参考であり、投資判断および結果はすべてユーザー自身の
責任に帰属します。当社は、本サービスの利用により生じたいかなる損害についても
責任を負いません。
```

### 6.3 禁止事項

- 大量自動アクセス (DOS的)
- リバースエンジニアリング・スクレイピング
- 他者の個人情報・資料の無断アップロード
- 著作権侵害

### 6.4 中立性宣言

```
当社は、不動産会社・仲介会社・販売会社・金融機関に対し、
個別の明示同意なしにユーザー情報・物件情報・分析履歴を提供しません。
物件提案・営業送客を目的とした事業を行いません。
```

### 6.5 終了・解約

- ユーザーはいつでも退会可能
- 退会時のデータ削除フロー
- 当社からの利用停止・終了条項 (規約違反等)

## 7. プライバシーポリシー (草案要点)

### 7.1 収集する情報

| 区分 | 例 | 利用目的 |
|---|---|---|
| アカウント情報 | メール、display_name | 認証 |
| 入力データ | 物件URL、Assumptions、アップロード資料 | 分析提供 |
| 行動ログ | ページビュー、クリック | サービス改善 |
| 決済情報 | Stripe customer ID | 課金処理 |
| 通信情報 | IP, UA, アクセス時刻 | セキュリティ |

### 7.2 利用目的

- 本人への分析提供
- サービス品質改善
- 匿名統計の作成・公開
- 不正利用検出
- 法令遵守

### 7.3 第三者提供

- **不動産会社・仲介会社・販売会社には提供しない**
- 法令に基づく場合のみ提供 (令状等)
- 業務委託先 (Supabase, Vercel, Fly.io, Anthropic, OpenAI, Stripe, Sentry, PostHog) との適切な契約

### 7.4 外部送信

- LLM API (Anthropic / OpenAI) への送信を明示
- マスク済みデータのみ送信
- 越境データ移転

### 7.5 開示・訂正・削除請求

- メール窓口
- 30日以内に対応

## 8. AI 出力の安全策

### 8.1 プロンプトレベル

すべての要約・質問生成プロンプトに以下を含める:

```
You are an analytical tool, NOT an advisor. NEVER use words like:
- "should buy" / "should sell" / "recommend" / "great deal"
- 「買うべき」「売るべき」「お得」「狙い目」「儲かる」

Use neutral framings:
- "with your assumptions" / 「ご指定の条件では」
- "appears expensive vs market" / 「市場対比で割高傾向」
- "DSCR is below the user's threshold" / 「DSCRがユーザー基準を下回る」
```

### 8.2 出力フィルタ

- NG ワードリストで post-processing
- 検出 → 警告ログ + 自動再生成
- 再生成も NG なら fallback テンプレート

### 8.3 監査ログ

- 全 LLM 呼び出しを model / prompt_version / output_hash で記録
- 月次レビュー (NG出力傾向の検出)
- ユーザー報告フォーム (アプリ内)

## 9. 税務 (個人事業主として)

### 9.1 整備事項 (青写真 2.2)

- 開業届 (Phase 0 中)
- 青色申告承認申請 (Phase 0 中)
- 会計ソフト (freee / マネーフォワード / 弥生)
- 事業用銀行口座 / クレカ
- 複式簿記
- 証憑保存 (AI利用料、サーバー費、ドメイン、法務、デザイン等)
- 事業実態の証拠 (LP、規約、GitHub、βユーザー、課金導線)

### 9.2 注意

- 個人事業の **赤字 × 給与所得の損益通算** が認められるかは事業所得性の実態次第
- 雑所得扱いされると赤字の損益通算は困難
- **税理士確認必須**

### 9.3 法人化のタイミング

- 月商安定 / 利益が出始めたら検討 (青写真 2.1)
- 損益通算メリットがなくなったタイミング
- 信用力 (BtoB対応・金融機関) が必要になったとき

## 10. その他のリスク

### 10.1 競合からのクレーム

- ポータルサイトから「うちのデータを使うな」と来た場合 → 該当ホスト kill switch
- スクレイピング訴訟リスクは allowlist + 手動入力でほぼ回避

### 10.2 ユーザーからの「儲からなかった」クレーム

- 利用規約 §6.2 で投資結果は自己責任を明記
- 「参考試算」を一貫表示
- スクリーンショットによる証拠保全

### 10.3 業者からの「うちの物件の評価が低い」クレーム

- 計算根拠 (engine_version + assumptions) で説明可能
- 評価方法をドキュメント化 (calculation_engine_spec.md 公開)
- 「分析者の前提に基づく試算」を明示

### 10.4 サービス停止リスク

- Supabase / Vercel / Fly.io の障害 → 監視 + ステータスページ
- LLM API の障害 → フォールバックモデル
- 全面停止時はステータスページで告知、24時間以上の障害は返金検討

## 11. NG文字列の自動チェック (CI 推奨)

`.github/workflows/legal-check.yml` 想定:

```bash
# ユーザー向け表示テキストに NG ワードが入っていないかチェック
patterns=("買うべき" "おすすめ" "狙い目" "儲かる" "絶対" "確実" "保証" "推奨します")
for p in "${patterns[@]}"; do
  if grep -r --include="*.tsx" --include="*.md" "$p" apps/web/src/ docs/product/ docs/legal/; then
    echo "❌ NG word found: $p"
    exit 1
  fi
done
```

`docs/legal/` 内の例示はOK (ホワイトリスト) なので、grep除外パターンで調整。
