## 比較の見るべきポイント

1. **第一印象**: ユーザーが3秒で受ける印象 (信頼感? 営業色? モダン? 古風?)
2. **数値の読みやすさ**: KPI・テーブル・スコアの可読性
3. **情報密度**: 一画面に収まる情報量
4. **「業者送客しない」が伝わるか**: 営業ポータルっぽさがないか
5. **対象ユーザーとの相性**: 初心者〜プロ投資家のどの層に刺さるか
6. **ダーク対応の必要性**: 現在ライト4 + ダーク1。ダーク版も別途あった方がよいか
7. **モバイル対応のしやすさ**: 情報密度高すぎるとモバイル展開が辛い

## 決定 (2026-05-12)

**採用: 03 Bloomberg系 (データ端末スタイル)**

選定理由:
- 「業者送客しない監査ツール」というコンセプトと、データ密度の高い端末UIの相性が良い
- 営業ポータルとの視覚的差別化が最も強い
- ターゲット (中級〜プロ投資家、本業ITや金融職) の道具感への共感
- 数値・テーブル中心の画面構成と親和性が高い

他の4案 (`01_linear`, `02_freee`, `04_stripe`, `05_notion`) は候補として保存。LP・診断テスト・初心者向けオンボーディング画面など、結果画面とは性格の違う面で再利用する可能性あり。
