## 操作フロー

1. 抽出済み項目をテーブル表示、各行に信頼度バッジ
2. 「編集」「入力」をクリック → inline editor
3. 不足資料は一覧表示、依頼文面生成ボタン
4. 「分析を実行」→ Assumptions に組み立てて POST `/analyze`
5. 修正は `extraction_corrections` に保存 (将来のFT用)

## 採用コンポーネント

- ヘッダ: `Panel` (#E-...)
- フィールド行: 自前 (label + value + Badge + 編集ボタン)
- 信頼度バッジ: `Badge` (level=good/warn/bad)
- 警告セクション: `Panel` + 自前リスト
- アクションバー: `Btn`

## バッジ凡例

- `HIGH` (緑): confidence ≥ 0.85
- `MED` (黄): 0.5 ≤ confidence < 0.85
- `LOW` (赤): confidence < 0.5
- `INFERRED`: LLM推測値
- `MISSING`: 抽出失敗 / 文書になし

## データ依存

- GET `/api/extractions/{id}` → ExtractionRecord (validated_output, field_confidences)
- PATCH `/api/extractions/{id}` → user修正を保存
- POST `/api/extractions/{id}/run-analysis` → Assumptions組み立て→分析

## 法務

- 「業者が悪意」と断定せず、慣習・テンプレートの問題として描く (assumption_critique_v1.md)
- 修正値は user_corrections に保存、業者には提供しない
