## 操作フロー

1. プラン選択 → Stripe Checkout (Pro/Heavy)
2. 「7日間試す」→ Stripe trial period (subscription_status=trialing)
3. Strong Supporter → 別商品 (Stripe で recurring 1,000円等)
4. 投げ銭 → Stripe Payment Intent (任意金額)
5. 単発レポート → 個別チェックアウト

## 採用コンポーネント

- プランカード: `Panel` (4 cols)
- 価格: 自前 (大型タイポ)
- 機能リスト: 自前 (●bullet)
- CTA: `Btn` variant=primary
- FAQ: 自前 (Q/A交互)

## データ依存

- POST `/api/billing/checkout` { plan } → Stripe Checkout session URL
- POST `/api/billing/donate` { amount } → Payment Intent
- GET `/api/billing/portal` → Customer Portal URL

## 法務 (legal_risk_checklist.md §5)

- 特商法表記を必須リンクで配置
- 「投げ銭」を「応援課金」と説明
- 解約条件・返金条件を明示
- beta機能の不確実性
- 「業者にデータが渡らない」を FAQ に明示

## デザイン上の判断ポイント

- "最人気" バッジは Pro に出すか (営業臭)
- "7日間試す" の trial 採用 (Stripe で簡単)
- 月額表記のみ vs 年額割引も並列表示
- 法人向けプランの存在 (v0では出さない)
