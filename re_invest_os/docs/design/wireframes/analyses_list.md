## 操作フロー

1. テーブルクリック → `/analyses/{id}` (結果画面に保存済みフラグ付き)
2. 複数選択 → 「比較に追加」ボタン → `/comparison/{board_id}`
3. ヘッダーソート可 (Score, IRR, DSCR, Cap, 日付, 価格)
4. フィルタは URL パラメータと同期

## 採用コンポーネント

- ヘッダ・タブ: `CmdKey`
- テーブル: TanStack Table (将来) または自前
- Score列: `Badge` (level=good/warn/bad)
- フィルタチップ: 自前 (`Btn` variant=ghost にアクティブ状態)

## データ依存

- GET `/api/analyses?limit=20&offset=0&filter=...` → AnalysisRunSummary[]
- DELETE `/api/analyses/{id}` (削除)

## パフォーマンス

- 1ページ20件、無限スクロール or ページネーション
- 各行の Score は事前計算済み (analysis_runs と score_results を join)
