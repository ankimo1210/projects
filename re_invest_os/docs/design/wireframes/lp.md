## 操作フロー

1. ヒーローCTA「無料で1分析を試す」 → `/new` (登録不要、後で保存はサインアップ要)
2. 「ログイン」「無料登録」 → Supabase Auth UI
3. 「特商法」「規約」 → 静的ページ
4. 「Pro」 → 課金プラン詳細 → Stripe Checkout

## 採用コンポーネント

- ヒーロー: 自前 (大型タイトル)、サブテキスト
- 約束カード: `Panel` + `Badge`
- 機能カード3つ: `Panel` (3 cols)
- 比較表: `Row` をリスト化
- 料金カード: `Panel` + `Btn`

## デザイン上の判断ポイント

- LPは Bloomberg ストレート (硬派) or Stripe寄り (柔らかい) のどちらにするか
- ヒーローに物件写真を置かない方が "営業ポータル感" を消せる
- "Live ticker" は LP には出さない (アプリ感を後出しにする)

## データ依存

- `/api/sample/nishi-shinjuku` の結果を「サンプル」として埋め込み可
- ユーザー数・分析数の動的表示は β後

## 法務 (legal_risk_checklist.md §1.3 と整合)

- 使う表現のみ
- "業者送客しない" 宣言を 3 箇所以上 (LP / 規約 / アプリ内)
