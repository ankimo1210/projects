## 操作フロー

1. URL入力 → POST `/api/listings/from-url` → job作成 → 進行表示 → 完了で `/extraction/{job_id}/confirm` へ
2. ドラッグ&ドロップ → POST `/api/documents/upload` → 分類 → 抽出 → 確認画面
3. 「手動入力」→ `/new`

## エラーハンドリング (Tier 1-4, ai_document_extraction_spec.md §12)

- Tier1: URL取得失敗 → Btn「手動入力に切り替え」
- Tier2: 抽出で主要項目null → 確認画面でユーザー修正を促す
- Tier3: ジオコード失敗 → 近傍比較スキップ、分析は続行
- Tier4: LLMタイムアウト → 1回retry → フォールバックモデル → 手動入力誘導

## 採用コンポーネント

- カード3つ: `Panel` (vertical stack)
- ドロップゾーン: 自前 (`Panel` 内、点線ボーダー)
- 進行バー: 自前 (CSS gradient)
- Job リスト: `Row` 派生

## データ依存

- POST `/api/listings/from-url` (Phase 3)
- POST `/api/documents/upload` (Phase 3)
- GET `/api/jobs/{id}` (Phase 3 / WebSocket or polling)
