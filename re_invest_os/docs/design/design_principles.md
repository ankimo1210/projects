## 1. ポジショニング

re_invest_os は **「個人投資家のための買付前監査ツール」** であり、不動産ポータルや投資家マッチングサービスとは別カテゴリのプロダクトとして見せる。

ビジュアル戦略上の比較対象:

| カテゴリ | 例 | 我々の関係 |
|---|---|---|
| 不動産ポータル | 楽待 / 健美家 / SUUMO | **避ける**。営業色・派手な広告・煽り |
| 不動産投資管理 | プロパティ管理SaaS | 中間 |
| 会計・税務SaaS | freee / マネーフォワード / 弥生 | 信頼感は学ぶ |
| 金融データ端末 | **Bloomberg / TradingView / Stripe Dashboard** | **目指す**。データ・冷静・プロ感 |
| 監査・法務ツール | 監査法人内製ツール / 弁護士DM | 思想は学ぶ |

中心メタファー: **「投資家の手元に置く、業者の言い分を検算するためのターミナル」**

## 2. デザイン原則 (10条)

### P1. データを主役にする
数値・テーブル・スコアが画面の中心。装飾・イラスト・写真は補助。グラフは派手にせず、目盛と数値で勝負。

### P2. 営業色を排除する
赤い「お得」「狙い目」バッジ、グラデーション派手、煽り文言、カウントダウン、リコメンド表示は使わない。

### P3. プロの道具に見せる、しかし排他的にしない
ターミナル風UI (F1/F2/...) や略号 (DSCR/Cap/IRR) は出すが、すべてに hover で日本語フル説明が出る。初学者にも開かれている。

### P4. ダーク基調
ダークモードを **デフォルト** とする。長時間の数値読解で目が疲れない、データ画面の没入感が出る。ライトモードも実装するが優先度2。

### P5. 情報密度は高く、しかし破綻させない
1画面で価格・KPI・スコア・サマリー・テーブル・最大買付価格まで見せる。ただしグリッド線・タイポ階層・色の役割を厳格に守ることで読みづらさを防ぐ。

### P6. 色の役割を固定する
- **アクセント (オレンジ)**: アクション・選択中・主要数値
- **緑**: 良好・基準クリア・収益
- **赤**: 警告・基準未達・損失
- **黄**: 注意・要確認
- **グレー階調**: 通常テキスト・補助情報

色を装飾目的で使わない。**色 = 状態**。

### P7. タイポを2書体に絞る
- **JetBrains Mono**: 数値・コード・略号・ナビゲーション
- **Noto Sans JP**: 和文本文・見出し
両方とも可変ウェイト対応。3書体目は導入しない。

### P8. アニメーションは最小限
ローディング・フォーカス・状態遷移のみ。装飾アニメは禁止。トランジションは ≤150ms。

### P9. 中立性を視覚で担保する
- 「買い推奨」「お得」「割安」を直接表示しない
- スコアは "ANLY:68/100" のように **分析値であって判定でない** ことを示す
- ストレステスト失敗 (`FAIL`) は赤で見せるが、それは事実の表示であって「やめろ」ではない
- LPと結果画面の脚注に「投資判断はユーザー自身」を必ず置く

### P10. モバイルは詳細でなく要点を見せる
密度を上げる端末UIはモバイルと相性が悪い。MVP段階ではモバイルでは **スコア・3行サマリー・主要KPI4個・最大買付価格のみ** をシンプルレイアウトで表示。詳細画面はデスクトップ前提。

### 3.3 セマンティックカラー (ダーク/ライト共通の役割)

| トークン | ダーク | ライト | 用途 |

| `--info` | `#4a9eff` | `#2563eb` | 情報・リンク |

### 3.4 グラフ用パレット

データ系列の色順:
1. `#ff8800` (アクセント)
2. `#4a9eff` (info)
3. `#00d96b` (good)
4. `#a78bfa` (purple)
5. `#ec4899` (pink)
6. `#888888` (neutral)

ヒートマップ: 緑 → 黄 → 橙 → 赤 (DSCR や IRR の感応度マトリクスで使用)。

## 4. タイポグラフィ

### 4.1 書体

```css
--font-mono: 'JetBrains Mono', 'Roboto Mono', monospace;
--font-sans: 'Noto Sans JP', 'Inter', system-ui, sans-serif;
```

### 4.2 用途マッピング

| 要素 | 書体 | 例 |
|---|---|---|
| 数値・金額 | mono | `¥39,800,000` `4.80%` `1.18` |
| KPI ラベル | mono uppercase | `CAP RATE` `DSCR MIN` |
| ナビ・コマンド | mono | `F1 ANLY` `F2 COMP` |
| 略号・タグ | mono uppercase | `BAD` `WARN` `LIVE` |
| 本文・説明 | sans | 物件説明・確認質問・サマリー |
| 見出し | sans | "総合スコア" "主要指標" |
| 物件名 | sans | "西新宿レジデンス 504号" |

### 4.3 サイズスケール

```
xs   10px   注釈・タグ
sm   11px   テーブル本文
base 12px   標準本文
md   13px   セクションラベル
lg   14px   小見出し
xl   16px   見出し
2xl  20px   ページタイトル
3xl  24px   KPI値
4xl  32px   サマリーKPI
5xl  56px   メインスコア
```

数値は **必ず `font-variant-numeric: tabular-nums`** で表示し、桁が揃うようにする。

### 4.4 ウェイト

| ウェイト | 用途 |
|---|---|
| 400 | 本文 |
| 500 | 強調・ラベル |
| 700 | 数値・見出し・ナビアクティブ |

300 以下は使わない (ダーク背景で可読性が落ちる)。

## 5. レイアウト

### 5.1 グリッド・スペーシング

8px ベース。

```
--space-1: 4px
--space-2: 8px
--space-3: 12px
--space-4: 16px
--space-5: 20px
--space-6: 24px
--space-8: 32px
--space-10: 40px
--space-12: 48px
```

### 5.2 ブレークポイント

| サイズ | 幅 | 用途 |
|---|---|---|
| sm | ≥ 640px | モバイル横 |
| md | ≥ 768px | タブレット |
| lg | ≥ 1024px | ラップトップ |
| xl | ≥ 1280px | デスクトップ標準 |
| 2xl | ≥ 1536px | 大画面・ワイド |

メインの分析画面は **lg+** を想定。モバイルは要点ビュー (P10 参照)。

### 5.3 パネル構造

```
┌──────────────────────────────┐
│ [SECTION] PANEL TITLE   meta │ ← panel-header (uppercase, accent color)
├──────────────────────────────┤
│  body                        │
│                              │
└──────────────────────────────┘
```

角丸: 0 または 2px (ターミナル風)。シャドウは使わない。境界線で表現。

### 5.4 ボーダー半径

- パネル・カード: `0px` または `2px`
- ボタン: `2px`
- 入力: `2px`
- バッジ・タグ: `2px`
- スコアリング・KPI: `2px`

Stripe/Notion 系の柔らかい `8px-14px` は使わない。

## 6. コンポーネント原則

### 6.1 KPIカード

```
┌────────────────────┐
│ CAP RATE           │ ← uppercase, muted, 9px, letter-spacing 0.08em
│ 4.80%              │ ← mono 24px bold
│ MKT 5.20 / -0.40pt │ ← delta, 10px, color-coded
└────────────────────┘
```

### 6.2 テーブル

- ヘッダ: `surface-alt` 背景、`accent` 文字、uppercase
- 行: 交互背景は使わず、`border` で区切る
- 数値列: 右寄せ、tabular-nums
- ホバー: 行全体を `surface-alt` に
- 判定列: バッジ (`GOOD` / `WARN` / `BAD`) で

### 6.3 バッジ

```
┌──────┐
│ WARN │  ← 9px mono uppercase, accent bg, dark text
└──────┘
```

3種類のみ:
- `GOOD` → 緑bg / 黒text
- `WARN` → 黄bg / 黒text
- `BAD` → 赤bg / 白text

### 6.4 ボタン

| 種別 | スタイル |
|---|---|
| Primary | accent背景 / 黒text / 2px radius |
| Secondary | transparent / border / accent text |
| Ghost | transparent / accent text on hover |
| Danger | bad背景 / 白text |

幅は内容に合わせる。最低 32px 高さ、12px パディング。

### 6.5 入力フォーム

- ラベル: 上配置、mono uppercase 11px muted
- 入力欄: surface背景、border、focus時 accent border
- 数値入力は右寄せ、tabular-nums
- ヘルプテキスト: 11px muted、入力欄下

### 6.6 ナビゲーション

トップバー固定。Bloomberg 風コマンドキー:

```
[F1 ANLY] [F2 COMP] [F3 HIST] [F4 SCREENER] [F5 RPT]
```

アクティブはアクセント色のキー。キーボードショートカット (F1〜F12) で実際に切り替え可能にする (Pro機能)。

---

## 7. アイコン

`lucide-react` を採用。

- サイズ: 14px (テキスト内) / 16px (ボタン) / 20px (パネルヘッダ) / 24px (大型)
- 線幅: 1.5px (デフォルト)
- 色: 文脈に応じて (muted / accent / semantic)
- 絵文字は使わない (Notion系の和やかさは別軸)

---

## 8. トーン・コピー

### 8.1 使う表現

- 「参考試算」「前提条件付き」
- 「割安傾向」「割高傾向」
- 「要追加確認」「データ不足」
- 「税後CFが弱い」「DSCR耐性が弱い」「出口前提への依存が高い」
- 「あなたの基準では…」(投資家自身の判定)

### 8.2 使わない表現

- 「買うべき」「おすすめ」「狙い目」
- 「儲かる」「お得」「割安です (断定)」
- 「購入推奨」「見送り推奨」
- 煽り (「今だけ」「残り○件」「他のユーザーが○人見ています」)
- 過度な絵文字 / 顔文字

### 8.3 ナンバリングとプレフィックス

ターミナル風で:

- 分析: `#A-184` (analysis)
- ジョブ: `#J-2418` (job)
- 物件: `#P-0931` (property)
- 比較: `#C-007` (comparison)
- レポート: `#R-052` (report)

短くて、人間が読み上げられる。

### 8.4 数値表記

| 種別 | 表記 |
|---|---|
| 金額 (本体) | `¥39,800,000` |
| 金額 (略) | `¥39.80M` (Million) / `¥124k` (千) |
| 利率・パーセント | `4.80%` (常に小数2位) |
| 倍率 | `1.42x` |
| 期間 | `8.4Y` / `12mo` |
| 日付 | `2026-05-12` (ISO) / `09:42 JST` |
| 差分 | `+0.90` / `-0.40pt` (pt = percentage point) |

## 9. アクセシビリティ

- WCAG AA: 全テキストでコントラスト比 4.5:1 以上
- ダーク背景上のオレンジ `#ff8800` は AA 通過 (テスト済)
- フォーカスリング: accent 2px solid、見える
- キーボード操作: ナビ・テーブル・モーダルすべて Tab で到達可能
- スクリーンリーダ: 略号 (DSCR/IRR/Cap) に `aria-label` で日本語フル名
- 数値変動 (ティッカー) は `prefers-reduced-motion` で無効化可能

## 10. NG リスト (避けるべきデザイン要素)

| NG | 理由 |
|---|---|
| グラデーション (Stripe風 purple→blue) | プロ感が消える |
| 大きな丸み (radius 12px+) | 端末感が消える |
| 装飾アイコン (🎯 🚀 💰) | Notion風になり営業色が出る |
| カードのドロップシャドウ強め | Stripe/Wealthfront感 |
| 不動産写真をヒーローに大きく置く | ポータル感、感情的判断を促す |
| カウントダウンタイマー | 煽り |
| 「他のユーザーが○人見ています」表示 | 営業ポータル特有の煽り |
| ライム/シアン/マゼンタなど派手色 | 端末感が消える |
| 3D 効果・skeuomorphism | 古さ・過剰装飾 |
| 自動再生アニメ | データ画面と相性悪い |

## 11. 実装ガイドライン

### 11.1 技術選定

- **Tailwind CSS v4** をベース
- **shadcn/ui** をテンプレートに、カラー・タイポをカスタマイズ
- **Recharts** または **Tremor** でチャート (端末感を出すためカスタムテーマ)
- **lucide-react** アイコン
- **Geist UI** や Catalyst は使わない (柔らかすぎる)

### 11.2 CSS変数の管理

`packages/shared-schemas/tokens.css` に集約 (次のステップで作成)。Next.js側は `globals.css` で `@layer base` に読み込む。

### 11.3 命名規則

- CSSクラス: Tailwind ユーティリティを基本に、必要なら `@layer components` でカスタム
- カスタムコンポーネント: `panel-header` / `kpi-cell` / `offer-cell` などセマンティック名
- ステート: `--data-state=active|hover|disabled` (HTML data属性) で表現

### 11.4 ダーク/ライト切り替え

`prefers-color-scheme` + 手動トグル。トグルは `localStorage.theme` に保存。
切り替えはアニメーションなし (画面のチラつきを避ける)。

## 12. 例外と LP の扱い

LP (ランディングページ) は分析画面ほど密度を上げない方が訴求力が上がる可能性がある。

候補:
- LP のみ **04 Stripe系** に寄せて柔らかく、サインアップ後の管理画面で **03 Bloomberg系** に切り替える
- もしくは LP も Bloomberg 統一で「最初の3秒の硬派さ」をブランド化する

決定は LP モック作成時に再検討する。
