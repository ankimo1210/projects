Methodology & integrity

  • 1-day execution lag — weights decided at the close of day t are held from t+1, so a signal built from day-t data cannot capture day-t's return (no same-day-close lookahead).
  • Transaction costs — 5 bps charged on turnover (Σ|Δw|); churn is penalized automatically.
  • Leverage caps — gross Σ|w| ≤ 1, per-name |w| ≤ 0.5; enforced by the engine, not the strategy.
  • Missing-asset handling — pre-IPO names (e.g. META before 2012-05) get weight 0 and the rest renormalize; blocks NaN-mediated lookahead.
  • Held-out segments — the headline Sharpe is measured only on the out-of-sample test window (2021-01-01..2025-06-30); the lockbox (2025-07-01→) is withheld until an explicit reveal.
  • Read-only metric — prepare.py is never edited by the loop; the agent only edits strategy.py's generate_weights.

Caveats (read honestly)

  • The Magnificent-7 is a hand-picked set of survivors, so absolute Sharpe levels are optimistic — this is not deployable alpha.
  • A high test Sharpe with a large train→test gap or a deeply negative worst rolling window is fragile/regime-fit; read the rolling and annual panels.
  • The lockbox bounds multiple-testing overfit accumulated across many experiments, but does not eliminate it.
  • The point of the demo is the autonomous search loop + cheat-proof metric, ported from Karpathy's autoresearch.
